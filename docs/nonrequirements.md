# 非機能要件（non-requirements）

## 1. 実行環境 / プラットフォーム
- Azure Functions を利用する
- プラン: Consumption Plan
- リージョン: Azure 東日本リージョン

## 2. 技術スタック
- 開発言語: Python 3.11

## 3. セキュリティ
- アクセス: 匿名アクセスを許可（認証・認可は不要）
- CORS: 不要

## 4. 可用性 / 信頼性
- 明示的な SLA / SLO は本要件では定めない
- エラー時でも HTTP 200 を返し、本文に `ERROR: ...` を返す（機能要件に従う）

## 5. 性能
- 目標応答時間、同時接続数、スループット等の数値要件は本要件では定めない
- 想定処理は整数演算（掛け算・割り算）であり、計算量は軽微

## 6. 運用 / 監視
- 監視・ログ収集（例: Application Insights）の採用有無やログ保存期間は本要件では定めない

## 7. 保守性 / 変更容易性
- API は 2 本（`/multiply`, `/divide`）のシンプルな構成を前提とする
- 将来拡張（新しい演算追加、レスポンス JSON 化等）は本要件の対象外

## 8. テスト
- テスト範囲、カバレッジ目標、CI 実行要件は本要件では定めない

## 9. ドキュメント
- 本リポジトリでは、機能要件は `docs/requirements.md`、非機能要件は本ファイルに記載する
